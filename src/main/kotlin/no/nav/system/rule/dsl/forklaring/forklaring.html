Regelverkspesifikasjon & Uttrykkstre

⚖️ Juristperspektivet

Sikre sporbarhet og etterprøvbarhet:

Hver enkelt beregning må kunne forklares juridisk gyldig tilbake til lovens bokstav, med referanser til konkrete lovhjemler (ID-ene som "SLITERTILLEGG-AVKORTING-TRYGDETID").

💻 Utviklerperspektivet

Transformere lovtekst til eksekverbar kode:

Kode som både regner riktig OG produserer full dokumentasjon automatisk - fra inngangsverdier via alle mellomberegninger til sluttresultat med hele uttrykk- og verdikjeden synlig.

📜 Lovtekst
Folketrygdloven § 20-18
→
🎯 Regelspesifikasjon

→
🔧 Eksekverbar Kode

→

etterlevelse 115.1

✨ Essensen

Samme kodesystem gir både riktig tallsvar og juridisk forsvarlig forklaring
— ingen gap mellom juss og kode

Eksempel fra koden:

// Lovhjemmel: § 20-18 Beregning av slitertillegg fun beregnSlitertillegg(...) = tracked { (fulltSlitertillegg() * trygdetidFaktor(...) * justeringsFaktor(...)) .navngi("slitertillegg") .id("SLITERTILLEGG-BEREGNET") ← Lovhjemmel-ID } // Gir automatisk: // 1. Beregning: 2291.67 kr // 2. Forklaring: "fulltSlitertillegg * trygdetidFaktor * justeringsFaktor" // 3. Uttrykkstre: Hele verdikjeden visualisert // 4. Sporbarhet: Referanse til SLITERTILLEGG-BEREGNET
